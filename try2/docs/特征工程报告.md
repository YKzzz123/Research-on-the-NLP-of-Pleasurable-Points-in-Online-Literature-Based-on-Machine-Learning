# 爽点段落 NLP 特征工程说明报告

## 1. 目的与数据流

本报告说明 `try2/build_nlp_features.py` 从标注语料到机器学习可用特征矩阵的完整流程。目标是为后续 **SVM、随机森林** 等传统模型提供 **全数值、无缺失、可复现** 的特征表，并与标签 `label_tag`（二分类：1 / 2）对齐。

**输入**：`try2/merged.csv`  

- 列：`book`（书名）、`chapter`（章节标题）、`paragragh`（爽点段落正文）、`tag`（已映射为 `label_tag` 的 1/2）。  

**运行环境**：建议在 **conda 环境 `comp5572`** 中执行（依赖 `jieba`、`numpy`、`scipy`、`scikit-learn`、`snownlp` 等与项目 `requirements.txt` 一致）。

```text
conda activate comp5572
python try2/build_nlp_features.py
```

---

## 2. 特征体系总览

特征可分为六类，按脚本中拼接顺序如下（不含标签）：

| 类别 | 维度（约） | 说明 |
|------|------------|------|
| 书名 one-hot | 5 | 五部作品各一维，取值为 {0,1} |
| 统计与表层 | 12 | 长度、句数、标点密度、小词典命中等 |
| 情感 | 2 | SnowNLP 文档情感与句间情感波动 |
| 字符 TF-IDF | 128 | sklearn 字符 2–4gram，带 L2 归一化 |
| 词级 TF-IDF | 96 | jieba 词 + 相邻词对（`词A::词B`） |
| LSA（截断 SVD） | 32 | 在「字符 TF-IDF ⊕ 词 TF-IDF」稀疏矩阵上的主题维 |

**训练用特征总维数**（含书名 one-hot）：**273**；再加一列 **`label_tag`** 即为监督标签。

---

## 3. 各类特征定义

### 3.1 书名 one-hot（5 维）

对 `book` 做 **独热编码**，列名形如 `book_大主宰`、`book_元尊` 等（具体顺序以运行时的 `sorted` 结果为准，并写入 `NLP_Feature_train_columns.json`）。  

**作用**：控制不同作品在文风、长度上的系统差异，避免模型把「书籍身份」与标签虚假关联时，至少显式给出可学习的区分信号。

---

### 3.2 统计与结构特征（12 维）

| 列名 | 含义 |
|------|------|
| `char_len` | 段落字符数 |
| `word_count` | jieba 分词后的词数 |
| `n_sentences` | 按 `。！？` 切分得到的句子数（至少为 1） |
| `mean_sentence_char_len` | 各句字符长度的均值 |
| `exclaim_per_1k` | 每千字中 `!` 与 `！` 的个数 |
| `question_per_1k` | 每千字中 `?` 与 `？` 的个数 |
| `lex_ugc_hits` | 与「碾压、震惊、骇然……」等小词表命中 **词种数**（jieba 词表级匹配） |
| `lex_poss_hits` | 与「得到、机缘、暴涨……」等词表命中词种数 |
| `lex_ugc_density` | `lex_ugc_hits / word_count` |
| `lex_poss_density` | `lex_poss_hits / word_count` |

词表为**人工归纳的提示词集合**，非穷尽；未命中时多为 0，属正常现象。

---

### 3.3 情感特征（2 维）

| 列名 | 含义 |
|------|------|
| `sentiment_snownlp` | 对**整段**正文调用 `SnowNLP(text).sentiments`，取值约在 **0～1**，数值越高表示模型越偏向「积极」评价（SnowNLP 基于其自带语料，与网文语境不完全一致，**宜作相对特征，非绝对情绪真值**）。 |
| `sentiment_sent_std` | 将正文按句切分后，**均匀抽样至多 8 句**分别算句级情感，再求标准差，刻画**段落内情感起伏**；单句或抽样不足时退化为 0。 |

抽样用于控制单次跑全量数据的耗时；若需更细粒度，可在脚本中增大抽样句数（会线性增加时间）。

---

### 3.4 字符 TF-IDF（128 维）

- **实现**：`sklearn.feature_extraction.text.TfidfVectorizer`  
- **参数要点**：`analyzer="char"`，`ngram_range=(2, 4)`，`max_features=128`，`min_df=2`，`sublinear_tf=True`，`norm="l2"`。  

列名在 CSV 中为 `char_tfidf_0` … `char_tfidf_127`；与具体 **n-gram 字符串** 的对应关系见 **`NLP_Feature_vocab.json`** 中的 `char_tfidf` 列表。

---

### 3.5 词级 TF-IDF（96 维）

- **分词**：`jieba.lcut`；**词条**包括：单词 + 相邻词对，词对拼接为 `词A::词B`（因自定义 `analyzer` 时 sklearn 不再使用内置 `ngram_range`，故在分析器内手工构造二元组合）。  
- **参数要点**：`max_features=96`，`min_df=2`，`sublinear_tf=True`，`norm="l2"`。  

列名 `word_tfidf_0` …；与词项对应关系见 **`NLP_Feature_vocab.json`** 中的 `word_tfidf`。

---

### 3.6 LSA / 截断 SVD（32 维）

- 将 **字符 TF-IDF 与词 TF-IDF** 的稀疏矩阵 **水平拼接** 后，对拼接结果做 **`TruncatedSVD`**（`random_state=42`），分量数取  
  `min(32, 特征数-1, max(2, n_samples//3))`。  
- 输出列名 `lsa_0` … `lsa_{k-1}`，得到稠密、相关性较低的 **「主题式」** 特征，有利于缓解纯高维稀疏 TF-IDF 对样本量较小时的方差问题。  

`NLP_Feature_vocab.json` 中 `svd_topics` 字段记录分量数及解释方差比之和（便于报告与调参）。

---

## 4. 数值安全与缩放策略

- 所有写入特征矩阵的数组均经 **`np.nan_to_num`** 处理，将 **NaN / Inf** 置为 **0**，避免 sklearn 报错。  
- **`NLP_Feature_train.csv`**：原始尺度（TF-IDF 已为 L2 归一化；树模型可直接使用）。  
- **`NLP_Feature_train_scaled.csv`**：对 **除书名 one-hot 外的连续特征** 做 **`StandardScaler`（零均值单位方差）**；**one-hot 列保持 0/1**，避免对离散示性变量错误缩放。  

**建议**：线性核 SVM、逻辑回归等优先使用 **`NLP_Feature_train_scaled.csv`** 或 **`Pipeline(StandardScaler(), 模型)`** 在交叉验证各折上单独 `fit`，以防信息泄漏。

## 5. 产出文件清单

| 文件名 | 说明 |
|--------|------|
| `NLP_Feature_Matrix.csv` | 含 `idx`、`book`、`chapter`、全部特征与 `label_tag`，便于与原文对照 |
| `NLP_Feature_train.csv` | **仅数值** + `label_tag`，推荐 **随机森林 / 梯度提升** 等 |
| `NLP_Feature_train_scaled.csv` | 连续特征已标准化，**线性 SVM / 逻辑回归** 等 |
| `NLP_Feature_vocab.json` | TF-IDF 列与 n-gram / 词项的映射；SVD 元信息 |
| `NLP_Feature_train_columns.json` | 书名 one-hot 顺序、特征列顺序说明 |

若 `NLP_Feature_Matrix.csv` 被占用无法覆盖，脚本可能自动写入带 `_run` 后缀的备份文件，以实际生成文件为准。

## 6. 局限与改进方向

1. **SnowNLP**：训练语料与网文差异大，情感分数**仅作辅助特征**；可替换或补充为基于句向量的无监督表征或领域微调模型。  
2. **小词典 `lex_*`**：覆盖有限，命中稀疏；可扩展词表或改为子串/正则统计。  
3. **样本量**：全量约数百条，高维 TF-IDF 易过拟合；建议 **交叉验证**、**特征选择** 或 **更强正则**。  
4. **书名 one-hot**：若测试集出现未训练过的书名，需统一编码策略（如训练时固定类别列表、未知书映射到全零或单独一类）。

---

## 7. 与标注任务的关系

标签 `label_tag` 来自合并标注规则（如：原多类映射为两类：偏「扮猪/优越感」为 1，偏「占有/机缘」等归为 2，具体以 `merge_annotations.py` 为准）。特征工程 **不修改标签**，仅基于 `paragragh` 与 `book` 构造输入；章节名 `chapter` 仅出现在完整矩阵中，**不进入** `NLP_Feature_train*.csv`，避免高基数类别列污染通用数值接口。

## 8. 特征选择与相关度分析

在生成 `NLP_Feature_train.csv` 后，可运行 **`try2/feature_selection_analysis.py`**（同样建议在 `comp5572` 环境中执行），用于：

| 指标 | 含义 |
|------|------|
| 与 `label_tag` 的 **Pearson 相关** | 单特征与二分类标签的线性相关程度；取绝对值可比较「谁与标签更相关」。 |
| **互信息 mutual_info** | 非线性依赖，对阈值化、稀疏特征较友好。 |
| **f_classif / p 值** | 单因素 ANOVA 思路，F 越大通常组间差异越明显。 |
| **score_combined** | 将互信息、\|相关\|、F 值分别 **秩归一化** 后取平均，作综合排序（可在脚本中改权重）。 |
| **特征间 \|r\| ≥ 0.88** | 记入 `feature_redundancy_pairs.csv`，提示两列可能冗余，建模时可择一保留。 |

**产出文件（默认 `try2/`）**

| 文件 | 说明 |
|------|------|
| `feature_scores.csv` | 每个特征的相关、互信息、F 值、p 值、`score_combined` |
| `feature_redundancy_pairs.csv` | 高相关特征对 |
| `selected_feature_list.json` | 按 `score_combined` 取 **Top 80**（`TOP_K` 可在脚本中修改） |
| `NLP_Feature_train_selected.csv` | 仅保留上述 Top-K 特征 + `label_tag`，便于降维训练 |
| `corr_label_top_features.png` | 与标签相关最高的特征柱状图 |
| `corr_feature_heatmap_top.png` | 与标签相关较高的特征之间的相关热力图 |

```text
python try2/feature_selection_analysis.py
```

**注意**：综合得分与相关度分析均在**全数据**上计算，用于探索与基线子集；若做严格测试集评估，应在训练折内重复特征选择或使用 `Pipeline`/`SelectKBest` 嵌套进交叉验证，避免信息泄漏。
