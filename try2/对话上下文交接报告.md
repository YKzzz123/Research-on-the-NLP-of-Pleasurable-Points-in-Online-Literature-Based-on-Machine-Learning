# 对话上下文交接报告（供下一轮会话继承）

## 一、目标与当前状态

本轮对话围绕「网文爽点二分类项目」完成了从**标注清洗 -> 特征工程 -> 特征选择 -> 模型训练 -> 展示平台**的一整套落地。

当前已具备：

- 可训练的特征矩阵（含情感特征）；
- 特征相关度/冗余分析产物；
- SVM + 随机森林 + 软投票器训练脚本与模型文件；
- Streamlit Demo（输入文本，输出 tag1/tag2/none + 爽度分数）。

## 二、关键需求变更与处理结果（按时间顺序）

### 1) 标注文件预处理与合并（try2）
- 用户要求：
  1. xlsx/xls 转 csv；
  2. `dazhuzai_ann.xlsx` 用 `raw_books/row_text/dazhuzai_all.txt` 匹配章节；
  3. tag 汇总为两类（后续更正“金手指归类为2”）；
  4. `yuanzun_ann.xlsx` 按章节号映射原文章节名；
  5. 合并 csv 输出列为 `chapter, paragragh, tag`；
  6. 产物放在 `try2`。

- 已实现脚本：`try2/merge_annotations.py`
  - 生成：
    - `dazhuzai_ann.csv`
    - `yuanzun_ann.csv`
    - `doupo_ann.csv`
    - `wanxiang_ann.csv`
    - `wdqk_ann.csv`
    - `merged.csv`
  - 后续增强：
    - 增加 `book` 列（现在为 `book, chapter, paragragh, tag`）；
    - 清洗 `【...】` 及内容；
    - 清洗 “第x更/第二更”等字样；
    - 大主宰章节匹配从简单前缀匹配升级为多片段+滑窗+全文回退，未匹配从 7 降到 0。

### 2) 标签规则调整
- 初版把“金手指”归在 1；
- 用户更正：**“金手指应归类为 2”**；
- 已修复：
  - `merge_annotations.py` 的 `normalize_tag` 中 `金手指 -> 2`；
  - `build_nlp_features.py` 的占有/收获侧词表纳入“金手指”。

### 3) 特征工程（可训练化）
- 脚本：`try2/build_nlp_features.py`
- 关键点：
  - 输入：`merged.csv`；
  - 输出：
    - `NLP_Feature_Matrix.csv`（含 idx/book/chapter，便于检查）
    - `NLP_Feature_train.csv`（纯数值 + label_tag，适合 RF）
    - `NLP_Feature_train_scaled.csv`（连续特征标准化，适合线性 SVM）
    - `NLP_Feature_vocab.json`
    - `NLP_Feature_train_columns.json`
  - 特征包含：
    - 统计特征（长度、句数、标点密度、词典命中）
    - SnowNLP 情感：`sentiment_snownlp` + `sentiment_sent_std`
    - char TF-IDF（2~4gram）
    - word TF-IDF（jieba unigram + bigram token）
    - LSA（TruncatedSVD）
    - book one-hot
  - 训练维数：约 273（含 one-hot，不含 label）。

### 4) 特征选择/相关度分析
- 脚本：`try2/feature_selection_analysis.py`
- 输出：
  - `feature_scores.csv`（相关、MI、F 值、综合分）
  - `feature_redundancy_pairs.csv`（强相关冗余对）
  - `selected_feature_list.json`
  - `NLP_Feature_train_selected.csv`（Top-K 默认 80）
  - `corr_label_top_features.png`
  - `corr_feature_heatmap_top.png`
- 备注：图已生成，但曾出现 matplotlib 中文字体缺失告警（不影响 CSV 结果）。

### 5) 展示平台与模型训练
- 新增脚本：
  - `try2/train_text_models.py`
  - `try2/demo_app.py`
  - `try2/text_features_common.py`（公共分词函数）
- 训练模型：
  - SVM（linear, prob）
  - RandomForest
  - VotingClassifier(soft)
- 训练输入：直接使用 `merged.csv` 的 `paragragh` + `tag`
  - 文本特征：char TF-IDF + word TF-IDF（共享分词函数）
  - 划分：`train_test_split(..., stratify=y, test_size=0.2)`
  - none 阈值：验证集正确样本最大概率的 10% 分位（当前约 0.6591）
- 训练产物：
  - `try2/model_eval.csv`
  - `try2/model_eval_report.txt`
  - `try2/artifacts/`
    - `char_vec.joblib`
    - `word_vec.joblib`
    - `svm_model.joblib`
    - `rf_model.joblib`
    - `voting_model.joblib`
    - `meta.json`

## 三、关键问题与修复记录（重要）

1. **自定义 analyzer 序列化加载失败**
   - 症状：`AttributeError: Can't get attribute 'zh_word_unigram_bigram' on <module '__main__'>`
   - 原因：训练脚本中函数定义在 `__main__`，joblib 反序列化时找不到。
   - 修复：提取到 `try2/text_features_common.py`，训练和 demo 都从该模块导入。
   - 状态：已重训并验证可推理。

2. **CSV 文件被 IDE 占用导致 PermissionError**
   - 在特征工程脚本中已做 fallback（输出 `_run` 文件）。
   - 当前若要覆盖同名文件，请关闭对应已打开 CSV。

3. **环境兼容问题**
   - 早期 base 环境有 numpy/scipy/sklearn 二进制不兼容；
   - 现统一使用 `comp5572` 环境运行，已可正常训练与推理。

4. **requirements 写入尝试**
   - 由于路径状态异常（工具找不到 requirements），未做最终一致更新；
   - 实际环境已通过 pip 在 `comp5572` 安装 `streamlit` 并成功导入。
